# ASCII Mockups

This document outlines the visual structure of the core pages for the Muhassaba app based on the UX decisions made on 2026-06-11.
(The app UI itself is purely Arabic).

## 1. Day Screen Layout (اليوم)
Primary view for filling out the daily scorecard. Uses a flat list. Completed tasks move to the bottom.

```text
+------------------------------------+
| ☰                الدرجة: ٨/١٠      |
|  <  السبت  الأحد  الإثنين  الثلاثاء> |
|      ٧٠٪   ٥٠٪    ٨٥٪    --        |
+------------------------------------+
| ▼ الصبح                            |
| [ ] الجماعة الأولى                 |
| [ ] أذكار الصباح             [ 📖 ] |
+------------------------------------+
| ▼ الظهر                            |
| [ ] الجماعة الأولى                 |
| [ ] السنن الرواتب                  |
+------------------------------------+
| ▼ القرآن                           |
| [ ] قراءة جزء من القرآن    (١٢/٢٠) |
|  (تتوسع لتعرض شريط تمرير عند الضغط)|
+------------------------------------+
|                                    |
| ▶ تم إنجازها (٤)                   |
+------------------------------------+
| [ اليوم ]   [ الأسبوع ]   [ الأذكار ] |
+------------------------------------+
```

## 7. Groups — المجموعات
New surfaces for the groups feature (student ↔ supervisor/مشرف). Phase 1 ships single-group-per-student.

The "الانضمام" action appears ONLY when I'm not already a student in a group

### 7.2 Join Group (الانضمام)
Reached from the hub or a `muhassaba://join?code=…` deep link (code prefilled).

Code design:
- 12 chars, Crockford Base32 alphabet `0123456789ABCDEFGHJKMNPQRSTVWXYZ`
  (omits ambiguous I L O U; case-insensitive; fully URL-safe — no special chars).
- Displayed grouped as `XXXX-XXXX-XXXX`; the dashes are presentation only.
- ~60 bits entropy → ~10^18 combinations, so guessing is infeasible without rate-limiting.
  Abuse is bounded by the code's expiry + manual rotation.

```text
+------------------------------------+
|             الانضمام لمجموعة        |
|                                    |
|    أدخل كود الدعوة من المشرف        |
|                                    |
|   [ _ _ _ _ - _ _ _ _ - _ _ _ _ ]  |
|                                    |
|          [ انضمام ]                |
|                                    |
|   (الكود غير صحيح أو منتهي)        |
+------------------------------------+
```

### 7.5 Assign Flow (تكليف بعمل)
Reuses the existing Library Picker (step 1), then a recipients step with all-with-exclusions or specific selection (step 2).

```text
  STEP 1 — ماذا تُكلِّف؟          STEP 2 — لمن؟
+----------------------------+  +----------------------------+
|        اختر عملاً           |  |   تكليف: أذكار الصباح       |
| [🔍 ابحث في المكتبة...    ] |  +----------------------------+
|                            |  | (•) كل الطلاب (استثناء)    |
| أعمال مقترحة               |  | ( ) طلاب محددون            |
| + صلاة الضحى       (يومي)  |  +----------------------------+
| + أذكار المساء     (ذكر)   |  | [✓] محمد علي               |
| + قراءة الملك      (ليلي)  |  | [ ] خالد إبراهيم  (مستثنى) |
|                            |  | [✓] يوسف سعيد              |
| + [ إنشاء عمل مخصص ]       |  | [✓] أحمد محمود             |
```

### 7.9 Group Stats — Supervisor (إحصائيات المجموعة)
NOTE: Part of the **separate statistics feature/spec**, not the groups MVP.
Headline is the **median (P50)**, not the average — a few inactive students at 0%
can't drag it down. Paired with an explicit active/inactive split + inactive list,
and per-deed adoption (% of students who completed each assigned deed).

```text
| < رجوع       إحصائيات المجموعة     |
```
